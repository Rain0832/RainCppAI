## 五层架构

```
┌──────────────────────────────────────────────────────────────────────────┐
│                          客户端 (Browser / curl)                         │
└────────────────────────────────┬─────────────────────────────────────────┘

├──────────────────────────────────────────────────────────────────────────┤
│  ② 协议层 — HttpContext 状态机                                            │
│     REQUEST_LINE → HEADERS → BODY → COMPLETE                             │
├──────────────────────────────────────────────────────────────────────────┤
│  ③ 中间件层 — MiddlewareChain                                             │

│  ⑤ 业务层 — 13 个 RouterHandler                                           │
│  ┌───────────────────────────────────────────────────────────────────┐   │

│  │  │   ├── AIStrategy (策略模式 — 4 种模型)                         │   │
```

## 关键数据流

```
请求 → onMessage → HttpContext 状态机解析
     → onRequest → handleRequest → 中间件 before
     → Router::route → Handler::handle
         ├─ 同步 Handler → 填 resp → HttpServer 发送
         └─ 异步 Handler (AI/SSE)
               → setDeferred(true)
               → aiThreadPool_.submit(...)
               → 线程池执行 curl
               → runInLoop → conn->send
     → 中间件 after
```

## 线程模型

```
main thread
  └─ muduo EventLoop (mainLoop_)
       └─ TcpServer.start()
            ├─ IO 线程 1 (EventLoop)   ← 处理连接 1-N 的读写
            ├─ IO 线程 2 (EventLoop)   ← 处理连接 N+1-M 的读写
            ├─ IO 线程 3 (EventLoop)
            └─ IO 线程 4 (EventLoop)

aiThreadPool_ (8线程)
  └─ 执行 curl_easy_perform (阻塞 ~1-5s)
       └─ 完成后 conn->getLoop()->runInLoop(send)

RabbitMQThreadPool (2线程)
  └─ 消费 sql_queue → executeMysql
```
