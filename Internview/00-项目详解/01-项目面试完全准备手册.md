## 二、高频面试题 & 参考答案

### 📌 A. 项目介绍（必问）

**Q1: 请介绍一下你的项目**

> 我做的是一个 **C++17 实现的 AI 应用服务平台**。整个项目分为两层：
>
> **底层是自研的 HTTP 服务器框架**（HttpServer），基于 muduo 网络库的 Reactor 模型，支持 HTTP 解析（状态机）、路由系统（精确 + 正则双匹配）、中间件链（CORS 等）、Session 会话管理、MySQL 连接池、SSL/TLS 加密通信。
>
> **上层是 AI 聊天应用**（ChatServer），支持：
> - **多模型 LLM 对话**：通过策略模式 + 工厂模式 + 静态自注册，一键切换通义千问、豆包等模型
> - **RAG 检索增强生成**：对接阿里百炼知识库，结合外部知识回答问题
> - **MCP 工具调用**：实现了 Model Context Protocol 的两段式推理，模型自主判断是否需要调用本地工具
> - **图像识别**：ONNX Runtime + MobileNetV2 进行端侧推理
> - **语音合成/识别**：百度 TTS/ASR
> - **异步入库**：通过 RabbitMQ 将聊天记录异步写入 MySQL，实现流量削峰

**Q2: 为什么选择 C++ 来做这个项目？**

> 1. **性能优势**：C++ 在网络 IO 密集型场景下的性能天花板更高。muduo 基于 epoll 的 Reactor 模型，能在单机处理大量并发连接。
> 2. **学习深度**：用 C++ 从底层实现 HTTP 服务器，可以深入理解网络编程、内存管理、并发编程。
> 3. **与 AI 推理结合**：ONNX Runtime 的 C++ API 性能最优，适合做端侧推理。
> 4. **实际需求**：AI Agent 的后端基础设施往往用 C++ 实现，展示全栈能力。

### 📌 B. 网络编程 & HTTP 框架（服务端开发重点）

**Q3: 解释一下 Reactor 模型，你的项目是怎么用的？**

> **Reactor 模型**是一种事件驱动的 IO 处理模型，核心思想是：
> - 一个**主线程（main Reactor）**负责监听和接受新连接（accept）
> - 多个**工作线程（sub Reactor）**各自运行一个 EventLoop，负责已建立连接的读写事件
> - 每个 EventLoop 通过 `epoll_wait` 监听 fd 上的事件，回调对应的 Handler
>
> 在我的项目中：
> - `muduo::net::TcpServer` 内部实现了 One Loop Per Thread 模型
> - 主线程 accept 新连接，通过 Round-Robin 分发到子线程的 EventLoop
> - 每个连接绑定一个 `HttpContext`（存储解析状态），通过 `conn->setContext()` 挂载
> - 数据到达时触发 `onMessage` 回调 → 状态机解析 → 路由分发 → Handler 处理 → 发送响应

**Q4: 你的 HTTP 解析器是怎么实现的？**

> 我实现了一个**有限状态机**来逐字节解析 HTTP 请求：
> `kExpectRequestLine → kExpectHeaders → kExpectBody → kGotAll`
>
> 关键点：状态机解决 TCP 流式协议和 HTTP 分段结构之间的阻抗不匹配。

**Q5: 精确匹配和正则匹配路由是怎么设计的？**

> Router 采用**双策略匹配**：精确匹配 O(1) 用 `unordered_map`，正则匹配 O(n) 用 `vector<regex>`。先查精确再查正则。

**Q6: 说说你的中间件设计**

> 中间件采用**洋葱模型**：顺序执行所有 before → Handler → 逆序执行所有 after。

### 📌 C. AI Agent & 大模型（AI 方向重点）

**Q7: 什么是 RAG？你是怎么实现的？**

> RAG（Retrieval-Augmented Generation）是检索增强生成，核心思想是在调用 LLM 前先检索相关文档注入 Prompt。
> 优化方向：分块策略、向量数据库选型、混合检索、Reranking、Query 改写。

**Q8: 什么是 MCP？你怎么实现 Agent 的工具调用？**

> 两段式推理：第一段意图识别（Prompt 注入工具列表让 LLM 判断），第二段结果综合（工具结果回注 Prompt）。

**Q9: 策略模式在你项目中是怎么用的？**

> 4 个 AIStrategy 实现 + 自注册工厂。利用 C++ 静态变量在 main() 之前初始化的特性自动注册。

**Q10: 多会话管理是怎么做的？**

> 二级 Map：`unordered_map<userId, unordered_map<sessionId, AIHelper>>`。

---

### 📌 D. 基础架构 & 中间件

**Q11: 数据库连接池是怎么设计的？**

> 单例 + queue + mutex + condition_variable + 自定义删除器归还连接 + 后台心跳检测。

**Q12: RabbitMQ 异步入库的设计思路？**

> MQ 解耦：Handler → MQ publish（~1ms）→ 后台消费者线程执行 INSERT。削峰填谷 + 主线程无阻塞。

**Q13: SSL/TLS 怎么集成的？**

> 自定义 BIO 回调桥接 muduo 非阻塞 IO 和 OpenSSL 阻塞 IO。

---

### 📌 E. C++ 八股文高频题

**Q14: shared_ptr vs unique_ptr？** 共享 vs 独占所有权。
**Q15: 可变参数模板？** 参数包 + 递归展开，编译期类型安全。
**Q16: runInLoop vs queueInLoop？** 条件异步 vs 始终异步的跨线程任务投递。
**Q17: RAII？** 构造获取资源 + 析构释放资源。
**Q18: std::move？** 类型转换 `static_cast<T&&>`，真正移动在移动构造/赋值中。

---

### 📌 F. 系统设计 & 性能优化

**Q19: 如果要支持 10W QPS 怎么优化？**

> 网络层（SO_REUSEPORT）+ 协议层（对象池）+ 业务层（异步 AI 调用+缓存）+ 架构层（Nginx+Redis+限流熔断）。

**Q20: 线程安全问题？**

> chatInformation/onlineUsers/连接池/MQ 连接都已加锁。优化方向：读写锁、细粒度锁、无锁数据结构。
