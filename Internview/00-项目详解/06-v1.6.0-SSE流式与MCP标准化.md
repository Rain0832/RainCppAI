## 一、SSE 流式输出

### 背景

之前 AI 回复需要等待完整响应（1-10 秒），前端用打字机模拟"流感"。真正的 SSE 可以在模型生成每个 token 时立即推送到前端。

### 架构变化

```
Before（等待完整响应）:
  fetch('/chat/send') → 等待 5s → 一次性收到全部内容

After（SSE 流式）:
  fetch('/chat/send-stream') → 立即开始接收 → 每 token 立即渲染
  data: {"token":"你"}\n\n
  data: {"token":"好"}\n\n
  data: [DONE]\n\n
```

### 后端实现

**`AIHelper::chatStream()`**：
```cpp
// curl WRITEFUNCTION 回调逐 chunk 处理 SSE 数据
size_t AIHelper::StreamWriteCallback(void *contents, size_t size, size_t nmemb, void *userp) {
    // 解析 "data: {...}" 行
    // 提取 delta.content（OpenAI 兼容格式）
    // 调用 onChunk(token) 回调
}
```

**`ChatSseHandler`**：
```
POST /chat/send-stream
→ 发送 SSE 响应头（text/event-stream）
→ 线程池执行 chatStream()
→ 每个 token 通过 runInLoop 回写 conn
→ 最后发 data: [DONE]
```

**响应头**：
```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
```

### 前端实现

```javascript
const response = await fetch('/chat/send-stream', {...});
const reader = response.body.getReader();
while (true) {
    const { done, value } = await reader.read();
    // 解析 SSE 帧，每个 token 立即渲染到 DOM
}
```

## 二、标准 MCP Server

### 背景

原实现是"伪 MCP"：把工具列表注入 Prompt，依赖 LLM 按 JSON 格式输出工具调用。真正的 MCP 是独立的服务端，通过 JSON-RPC 2.0 协议通信。

### 新实现

**JSON-RPC 2.0 接口**（路由：`POST /mcp`）：

```json
// 请求：tools/list
{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}

// 响应：
{"jsonrpc":"2.0","id":1,"result":{"tools":[
  {"name":"get_weather","description":"...","inputSchema":{...}},
  {"name":"get_time","description":"...","inputSchema":{...}}
]}}

// 请求：tools/call
{"jsonrpc":"2.0","id":2,"method":"tools/call",
 "params":{"name":"get_weather","arguments":{"city":"广州"}}}

// 响应：
{"jsonrpc":"2.0","id":2,"result":{"content":[{"type":"text","text":"..."}]}}
```

**`McpServer` 类**：
- 复用现有 `AIToolRegistry`（工具逻辑零重复）
- 额外维护 `toolMetas_`（name + description + inputSchema）
- 支持动态注册新工具（`registerTool()`）

### 兼容性

标准 MCP Server 可被任意支持 MCP 协议的 Host（Claude Desktop、Cursor 等）直接接入，不再依赖 LLM 理解 Prompt 格式。

## 四、面试考点

| 问题 | 核心答案 |
|------|---------|
| SSE vs WebSocket？ | SSE 单向（服务器推）、基于 HTTP、天然断线重连；WebSocket 双向、需要升级协议。AI 聊天单向推送用 SSE 更轻量 |
| SSE 数据格式？ | `data: <payload>\n\n`，每帧以双换行结束；`data: [DONE]` 表示流结束 |
| MCP JSON-RPC 2.0 核心字段？ | `jsonrpc: "2.0"`, `id`, `method`, `params`（请求）；`result` 或 `error`（响应）|
| 为什么用 fetch().body.getReader() 而不是 EventSource？ | EventSource 只支持 GET，无法发送请求体；fetch streaming 支持 POST |
| Stream curl 回调在哪个线程？ | 线程池线程（非 IO 线程），通过 `runInLoop` 发送到正确的 IO 线程 |
