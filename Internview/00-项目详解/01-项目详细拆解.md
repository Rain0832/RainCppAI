## 一、架构总览

### 1.1 五层架构

```
┌──────────────────────────────────────────────────────────────────────────┐
│                          客户端 (Browser / curl)                         │
└────────────────────────────────┬─────────────────────────────────────────┘
                                 │ HTTPS / HTTP
┌────────────────────────────────▼─────────────────────────────────────────┐
│  ① 网络层 — muduo TcpServer (Reactor, 4 IO线程, epoll)                   │
├──────────────────────────────────────────────────────────────────────────┤
│  ② 协议层 — HttpContext 状态机 (REQUEST_LINE → HEADERS → BODY → DONE)    │
├──────────────────────────────────────────────────────────────────────────┤
│  ③ 中间件层 — MiddlewareChain (洋葱模型: before → Handler → after)       │
├──────────────────────────────────────────────────────────────────────────┤
│  ④ 路由层 — Router (精确O(1) + 正则O(n))                                 │
├──────────────────────────────────────────────────────────────────────────┤
│  ⑤ 业务层 — 15 个 Handler + AI 线程池(8线程)                             │
│     ChatServer → AIHelper → AIStrategy(4模型) → curl → LLM API          │
│     McpServer → JSON-RPC 2.0 → AIToolRegistry                          │
│     ImageRecognizer → ONNX Runtime → MobileNetV2                        │
│     AISpeechProcessor → 百度 TTS/ASR                                    │
│     MQManager → RabbitMQ → MySQL (异步入库)                              │
└──────────────────────────────────────────────────────────────────────────┘
```

### 1.2 线程模型

```
muduo main thread → EventLoop → TcpServer
  ├─ IO线程 1-4 (epoll_wait → onMessage → 状态机解析 → 路由 → Handler)
  │
  ├─ AI线程池 (8线程)
  │   └─ chat() / chatStream() → curl_easy_perform → runInLoop → conn->send
  │
  └─ MQ消费线程 (2线程)
      └─ BasicConsumeMessage → executeMysql(sql)
```

### 1.3 数据流

**同步请求**（登录/注册/历史）:
```
IO线程: onMessage → 状态机 → Router → Handler → 填 resp → send
```

**异步请求**（AI 聊天）:
```
IO线程: onMessage → Handler → setDeferred(true) → 提交线程池 → 立即返回
线程池: curl 阻塞 → 拿到结果 → runInLoop → conn->send
```

**SSE 流式**（v1.6.0）:
```
IO线程: onMessage → ChatSseHandler → 发送 SSE 握手头 → 提交线程池
线程池: chatStream() → curl WRITEFUNCTION 每收到 token → runInLoop → conn->send("data: ...")
```

## 二、核心模块详解

### 2.1 多模型适配（策略 + 工厂 + 自注册）

```cpp
AIStrategy ←── AliyunStrategy ("1")     // 通义千问 qwen-plus
          ├── DouBaoStrategy ("2")       // 火山引擎豆包
          ├── AliyunRAGStrategy ("3")    // 百炼 RAG 知识库
          └── AliyunMcpStrategy ("4")    // 百炼 MCP 工具调用

// 静态自注册（main 前执行，无需手动 switch）
static StrategyRegister<AliyunStrategy> reg("1");
```

### 2.2 消息模型与线程安全

```cpp
struct Message {
    std::string role;    // "user" / "assistant"（v1.4.0 消除奇偶判断）
    std::string content;
    long long   ts;
};

class AIHelper {
    std::mutex          msgMutex_;    // 保护 messages_
    std::vector<Message> messages_;
    std::atomic<bool>   processing_;  // CAS 串行化同一 session
};
```

### 2.3 SSE 流式输出（v1.6.0）

```
POST /chat/send-stream
→ 发送 text/event-stream 头
→ AIHelper::chatStream() 用 curl WRITEFUNCTION 逐 token 回调
→ StreamWriteCallback 解析 OpenAI 兼容 SSE 格式（data: {"choices":[{"delta":{"content":"..."}}]}）
→ 每个 token 通过 runInLoop → conn->send("data: {\"token\":\"x\"}\n\n")
→ 最终发 data: [DONE]\n\n
```

### 2.4 标准 MCP Server（v1.6.0）

```
POST /mcp
→ JSON-RPC 2.0：
  tools/list → 返回 {name, description, inputSchema}
  tools/call → AIToolRegistry::invoke → 返回 {content:[{type,text}]}
```

### 2.5 数据库设计（v1.5.0 三高表）

| 表 | 核心字段 | 设计意图 |
|----|---------|---------|
| `users` | id(PK), username(UK) | 用户注册登录 |
| `sessions` | id(UUID PK), user_id, title, deleted_at | 会话持久化 + 软删除 |
| `messages` | id(AUTO_INCREMENT PK), session_id, role(ENUM), content | 消息自增序号 + 显式角色 |
| `user_api_keys` | user_id + provider(UK) | 用户级 API Key 存储 |

### 2.6 异步入库

```
AIHelper::addMessage → MQManager::publish("sql_queue", SQL)
                          ↓ (~0.1ms, 不阻塞)
                    RabbitMQ 队列
                          ↓
              2个消费线程 → executeUpdate(SQL)
```

### 2.7 并发安全体系

| 数据结构 | 锁类型 | 读/写 |
|---------|--------|------|
| chatInformation | shared_mutex | shared_lock 读 / unique_lock 写 |
| messages_ (AIHelper) | mutex | lock_guard |
| processing_ (AIHelper) | atomic<bool> | CAS compare_exchange |
| onlineUsers_ | shared_mutex | 同上 |
| DB 连接池 | mutex + cv.wait_for(3s) | 超时快速失败 |
| LRU (list+map) | 依附 chatInfo 写锁 | O(1) touch/evict |
