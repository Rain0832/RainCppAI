## 更新规则

### 每次优化后必须更新

1. **`00-项目详解/04-优化记录与STAR.md`** — 版本表新增一行
2. **`00-项目详解/02-高频面试题.md`** — 复习要点 checklist 补充新知识点
3. **本 README 的目录结构** — 若新增了文档
4. **对应知识域文件** — 如新增网络优化，更新 `02-网络基础/`

### 每次大版本（x.0.0）还需更新

- `00-项目详解/01-架构全景图.md` — 架构图是否变化
- 新建对应的 `0x-vX.Y.Z-XXX.md` 知识沉淀文章

### 版本号规则

| 变动 | 版本号 |
|------|--------|
| 大改动（架构级） | +1.0.0 |
| 中改动（多文件联动） | +0.1.0 |
| 小改动（单文件） | +0.0.1 |
