## 更新规则

### 每次代码优化后必须更新

1. **`00-项目详解/03-版本优化汇总.md`** — 版本总表新增一行 + 新版本详情
2. **`00-项目详解/01-项目详细拆解.md`** — 若架构/模块有变化
3. **`00-项目详解/02-面试介绍版.md`** — 若新增了面试高频考点
4. **本 README 的目录结构** — 若新增了文档文件
5. **对应知识域文件** — 如网络优化 → `02-网络基础/`

### 每次大版本（x.0.0）还需更新

- `01-项目详细拆解.md` 的架构图
- 新建版本知识沉淀文章（如果改动量大到需要单独说明）

### 版本号规则

| 变动 | 版本号 |
|------|--------|
| 大改动（架构级） | +1.0.0 |
| 中改动（多文件联动） | +0.1.0 |
| 小改动（单文件） | +0.0.1 |
